# Documentation Structure

## Main Document

**`REQUIREMENTS.md`** - This is the **single source of truth** for all project requirements. It consolidates all information from the original documents into one comprehensive document.

This document includes:
- Project vision and objectives
- Core system features (Phase 1 MVP)
- User interface design
- Technical stack
- System architecture
- Jordanian agencies
- Implementation phases
- Operational scenarios
- Non-functional requirements
- Future enhancements

## Archive

The `archive/` folder contains the original documents that were consolidated into `REQUIREMENTS.md`:

- `readme.md` - Original vision and functional overview
- `srs.md` - System Requirements Specification
- `Concept of Operations.md` - Operational scenarios and workflows
- `dual_web_screen.md` - UI design specifications
- `System Architecture Data.md` - Architecture and data flow
- `technology stack.md` - Technology recommendations
- `consolidated_requirements.md` - Initial consolidation attempt

These are kept for historical reference but are superseded by `REQUIREMENTS.md`.
